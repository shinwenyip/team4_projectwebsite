Critical Evaluations

User Interface and Experience

Our website has a navigation bar on the top of every page. Users can redirect to any page if they want to. Also, there is a page for community guildelines to ensure the community stays protected. Admin users can add as many guildelines as they want in admin page. Currently we have "Fake engagement policy" and "Impersonation policy" as examples guildelines. And we have a simple introduction about our website on the home page and every visitor and logged in user would be redirected to this page before they dive the website.



Functionality & Efficiency

We achieved all MUST and SHOULD, and most of COULD requirements in MoSCow list. And in order to keep user data safe and to test UCL/Google SSO, we deployed our website with HTTPS.

In order to schedule asynchronous task for challenge state to change to ended when the deadline is reached, We use a tool called celery to run the task outside the HTTP request-response cycle. And to pass messages between Django project and Celery workers, we use Redis (an in-memory key-value database) as a message broker. Besides, for increasing performance of leaderboard, we also used Redis as a cache backend to store and manage user scores.



Stability

Our project took rigorous and extensive testing procedure at the end of project. We did unit tests, functional tests and load tests. In unit tests, we reached 93% code coverage totally and had a good performance in load test which can read the report at https://bit.ly/3slW0f2.



Maintainability

Since we used the Django framework which has a very clear MTV (Models, Templates, Views) structure, it is not hard to learn and maintain the work pattern for our project. And we wrote all denpendencies needed in github src/requirements.txt.

Also, We uploaded a code review video in YouTube which contains the system architecture diagram, data flow diagram, code reviewing and user demo.



Project Management

We had at least one inner-team meeting every week with our team members. And in these meetings, we did preparing work for coding such as database constructing, requirements analysis etc., then we always made a plan for next week and allocated different tasks to individuals.

Besides, we also had one client meeting every week to present our progress and got feedback from them.
